# Висновки щодо швидкостей алгоритмів пошуку

## 1. Перший текст

**Результати вимірювань:**
- KMP: **0.339 с**
- Boyer–Moore: **0.105 с**
- Rabin–Karp: **0.679 с**

**Висновки для першого тексту:**
- **Boyer–Moore** показав **найкращу швидкість**, майже у 3 рази швидше за KMP і приблизно у 6 разів швидше за Rabin–Karp.
- **KMP** має стабільну середню продуктивність, але поступається Boyer–Moore.
- **Rabin–Karp** виявився **найповільнішим**, що може бути пов’язано з накладними витратами на хешування та можливими перевірками колізій.

Для цього тексту **Boyer–Moore є найбільш ефективним алгоритмом**.

## 2. Другий текст

**Результати вимірювань:**
- KMP: **1.644 с**
- Boyer–Moore: **0.260 с**
- Rabin–Karp: **3.132 с**

**Висновки для другого тексту:**
- **Boyer–Moore** знову демонструє **найвищу продуктивність**, випереджаючи KMP більш ніж у 6 разів, а Rabin–Karp — більш ніж у 12 разів.
- **KMP** суттєво сповільнився порівняно з першим текстом, що може свідчити про більший розмір тексту або менш сприятливу структуру для цього алгоритму.
- **Rabin–Karp** показав **значне падіння продуктивності**, ставши найгіршим варіантом для цього тексту.

Для другого тексту **Boyer–Moore є безумовним лідером за швидкістю**.

## 3. Загальні висновки

- **Boyer–Moore** є **найшвидшим алгоритмом у всіх тестах**, незалежно від тексту.
- **KMP** демонструє **стабільну, але середню продуктивність**: він значно швидший за Rabin–Karp, але помітно повільніший за Boyer–Moore.
- **Rabin–Karp** в обох випадках є **найповільнішим**, особливо на великих або складних текстах.

### Загальний рейтинг швидкості:
1. **Boyer–Moore** — найефективніший
2. **KMP** — компроміс між стабільністю та швидкістю
3. **Rabin–Karp** — найменш ефективний у даних експериментах
